# Part 1 — ALKU (Opening Credits)

> **Timeline**: 0s – 65s | **Original code**: WILDFIRE (Samuli Syvähuoko)
> **Source**: `ALKU/` folder in SecondReality repo | **Registry name**: `ALKU`

---

## Overview

ALKU ("Beginning" in Finnish) is the opening credits sequence. It displays
Future Crew team member names over a black background, then fades in a
scrolling landscape image while continuing to show credits.

The effect is deceptively simple — the visual impact comes entirely from
timing, anti-aliased text, and smooth palette-based fading, all
tightly synchronized to the opening bars of Skaven's soundtrack.

---

## How It Works

### Display Mode

- **Resolution**: 320×400 (custom VGA tweaked mode, 256 colors)
- **Landscape area**: Lines 50–449 (200 source lines, each doubled)
- **Frame rate**: 70 Hz (VGA vertical blank)

### The "Terrain" — It's Not 3D

The landscape is a **pre-rendered 640×200 pixel image** (`HOI.IN0` + `HOI.IN1`)
that scrolls horizontally. There is no heightmap, no raycasting, no 3D
rendering of any kind. The illusion of depth comes from the artwork itself.

- **Scroll speed**: ~7.8 pixels/second at 70 Hz
- **Total scroll distance**: 320 pixels (image is 640 px wide, viewport is 320 px)
- **Vertical doubling**: Each source line is drawn twice to fill the 400-line display

### Text Rendering

The font is a **variable-width bitmap** stored in `FONA.INC` (1500×30 pixels).
Each pixel has one of 4 values representing anti-aliasing levels:

| Font Value | Meaning | Alpha Equivalent |
|------------|---------|-----------------|
| 0 | Transparent | 0% |
| 1 | Light edge | ~33% |
| 2 | Medium edge | ~67% |
| 3 | Full text | 100% |

Characters are stored contiguously; boundaries are detected by scanning for
blank (all-zero) columns. Character order:
```
ABCDEFGHIJKLMNOPQRSTUVWXabcdefghijklmnopqrstuvwxyz0123456789!?,.:$#()+-*='[]
```

The `[` and `]` characters are special — they form the two halves of a
Dolby Surround logo.

### Palette-Based Anti-Aliasing

The original achieves text-over-background blending without per-pixel alpha:

1. **256 palette entries** divided into 4 banks of 64
2. **Bank 0** (indices 0–63): Pure landscape colors
3. **Banks 1–3** (indices 64–255): Landscape colors blended with text at
   3 opacity levels

Font pixels are `OR`ed into the framebuffer:
- Value 1 → `0x40` (bank 1, lightest text blend)
- Value 2 → `0x80` (bank 2, medium blend)
- Value 3 → `0xC0` (bank 3, full text blend)

The OR ensures each pixel's palette bank encodes both the background
color (low 6 bits) and the text opacity (high 2 bits). Palette switching
then controls whether text is visible.

### Fade Effects

All fading is done by palette interpolation:

- **Text fade**: Interpolate between `palette` (no text) and `palette2`
  (text visible) over 64 frames
- **Background fade**: Interpolate between `blackpalette` and `palette`
  over 128 frames

## WebGL Implementation

### Architecture

The effect uses two textures composited by a fragment shader:

1. **Landscape texture** (640×200 RGBA) — decoded from the original indexed
   pixel data at init time using the VGA palette
2. **Text textures** (320×256 RGBA each, 7 total) — pre-rendered at init time
   using the decoded bitmap font

### Rendering Pipeline

```
init(gl):
  1. Decode landscape base64 → indexed pixels → RGBA via palette → GL texture
  2. Parse font: decode base64, find character boundaries
  3. Pre-render 7 text screens to RGBA buffers → GL textures
  4. Compile compositing shader

render(gl, t, beat):
  1. Compute sequence from time → derive scroll, bgFade, textFade
  2. Bind landscape texture (unit 0) + current text texture (unit 1)
  3. Pass uniforms: uScrollOffset, uBgFade, uTextFade, uShowLandscape
  4. Draw fullscreen quad
```

## Remastered Ideas

The remastered version should preserve the core identity — serene credits
over a natural landscape — while leveraging modern capabilities:

### Visual Enhancements

- **Parallax landscape**: Split the flat image into 2–3 depth layers (sky,
  far hills, near foliage) with differential scroll speeds for a subtle
  parallax effect
- **Atmospheric fog**: Add distance-based fog/haze between parallax layers,
  with slight color shift toward the sky tone
- **Particle system**: Gentle floating particles (fireflies, pollen, dust
  motes) in the foreground, influenced by beat
- **Dynamic lighting**: Subtle color temperature shifts over time —
  dawn-to-morning feel as the landscape scrolls

### Text Enhancements

- **SDF text rendering**: Replace the bitmap font with signed distance field
  text for crisp rendering at any resolution
- **Text animation**: Characters could fade in individually (typewriter
  style) rather than all at once, with a subtle wave
- **Glow effect**: Soft bloom around text characters, intensity pulsing
  gently with the beat
- **Higher resolution**: 4K rendering with subpixel text positioning

### Audio Reactivity

- **Beat-synced fades**: Lock text fade timing precisely to bar boundaries
  rather than frame counting
- **Amplitude-driven particles**: Particle density/speed responds to
  audio amplitude
- **Bass-reactive landscape**: Very subtle vertical displacement of the
  landscape on bass hits

### Atmosphere

- **Film grain**: Light procedural noise overlay for a cinematic quality
- **Vignette**: Gentle darkening at screen edges
- **Color grading**: Desaturated, slightly warm tone reminiscent of 90s
  demo scene aesthetics
