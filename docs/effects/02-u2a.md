# Part 2 — U2A (3D Ships Flyover)

**Timeline:** 65.0 s – 78.0 s (13 seconds)  
**Original code:** PSI (Samuli Syvahuoko)  
**Original title:** "Alkutekstit II" (Opening Credits 2)  
**Source folder:** `VISU/`, `VISU/C/` in SecondReality repo  

## What It Does

Three polygon-based spaceship models fly over the viewer against the ALKU
landscape background. The camera sweeps along a pre-authored path while
objects appear/disappear and rotate through space. This is the first 3D
polygon scene in the demo, immediately following the ALKU text credits.

## How the Original Works

### Display Mode
VGA Mode 13h — 320×200, 256 colors, 70 Hz refresh rate.

### 3D Engine (VISU)
The scene uses Future Crew's proprietary "U2" 3D engine, shared between
U2A (ships) and U2E (city flyover, Part 22). Key components:

- **Object format:** FC's chunked binary with VERS, NAME, VERT, NORM,
  POLY, ORD0/ORDE sections. Vertices are 32-bit fixed-point ÷ 16384.
  Normals are 16-bit fixed-point ÷ 16384.
- **Polygon rendering:** Convex polygon scan-line fill with Sutherland-
  Hodgman clipping (Z near/far, then 2D up/down/left/right).
- **Shading:** Per-polygon flat shading or per-vertex Gouraud shading,
  selected by material flags. Uses a fixed directional light source
  `[12118, 10603, 3030] / 16384`.
- **Ordering:** Painter's algorithm (back-to-front sort by center vertex Z).
  Each object has multiple precomputed polygon draw orders; the best one
  is chosen based on the viewing angle's closest orientation point.

### Animation
The entire camera path and object transforms are stored in a packed
binary stream (`.0AB` file), decoded frame-by-frame at 70 fps:

- `0xFF + byte ≤ 0x7F`: Change field of view
- `0xFF + 0xFF`: Animation end
- `0xC0` mask: Select object by number
- `0x80 / 0x40`: Show / hide selected object
- Translation deltas: Variable-length signed values (1/2/4 bytes)
- Rotation matrix: 3×3 delta-compressed entries

Camera is object 0; it uses raw coordinate values while other objects
divide by 128 (fixed-point normalization). Initial FOV is 40°.

### Background
The ALKU landscape image at scroll position 320 (the end of ALKU's
scroll) serves as a static backdrop. Its palette is remapped to indices
192+ to avoid conflicts with the 3D object colors. The 3D rendering is
clipped to Y range [25, 174], leaving the sky area (0–24) black and the
lower landscape (175–199) visible below the ships.

### Projection
```
x_screen = x_3d × projXFactor / z_3d + 159
y_screen = y_3d × projYFactor / z_3d + 99
projXFactor = (319 − 159) / tan(fov / 2)
projYFactor = projXFactor × 0.86  (VGA pixel aspect correction)
```

## WebGL2 Implementation

### Architecture
The implementation is a faithful software rasterizer port — the U2 engine
runs entirely in JavaScript, rendering to a 320×200 indexed framebuffer.
Each frame, the indexed pixels are converted to RGBA via the palette and
uploaded as a WebGL texture for display.

**Files:**
- `data.js` — Base64-encoded 3D objects, animation stream, scene/palette
- `engine.js` — Self-contained U2 engine port (object loader, animation
  decoder, matrix math, projection, clipping, flat/Gouraud polygon fill)
- `effect.js` — WebGL wrapper (background prep, palette, texture upload)

### Per-Frame Pipeline
1. Clear framebuffer to black
2. Blit ALKU landscape background (lines 25–198)
3. Advance animation stream (variable frame count to maintain 70fps sync)
4. For each visible object: apply camera transform, project vertices
5. Sort objects back-to-front by center-point Z distance
6. For each object: select best polygon draw order, clip & rasterize
7. Convert indexed framebuffer → RGBA via palette lookup
8. Upload to GPU texture, draw fullscreen quad with beat pulse

### Display Mapping
The 320×200 render is stretched to fill the 320×256 display area via
texture sampling, which approximates the original CRT's non-square pixel
aspect ratio (Mode 13h pixels were taller than wide).

## Remastered Ideas

### Visual Enhancements
- **Higher-resolution models:** Subdivide the ship polygons or use
  smooth-shaded higher-poly versions while maintaining silhouette fidelity.
- **Phong shading:** Replace flat/Gouraud with per-pixel Phong for
  smoother specular highlights on the ship hulls.
- **Environment mapping:** Add subtle reflections of the landscape on
  the metallic ship surfaces.
- **Particle trails:** Engine exhaust particles trailing behind each ship,
  synchronized to the beat.
- **Motion blur:** Temporal accumulation or velocity-buffer blur for the
  fast-moving ships.
- **Depth of field:** Gentle blur on the background landscape with the
  ships in sharp focus.

### Background Enhancements
- **Parallax scrolling:** Separate the landscape into depth layers for
  a subtle 3D background effect as the camera moves.
- **Atmospheric haze:** Add distance fog that fades the landscape colors.
- **Dynamic lighting:** The ships cast shadows or light spills onto the
  landscape below.

### Animation Enhancements
- **Camera shake:** Subtle camera vibration synced to bass hits.
- **Lens flare:** Bright flash as ships pass close to the viewer.
- **Chromatic aberration:** Brief color fringing on fast camera moves.
- **Beat-synced FOV pulse:** Slight FOV zoom on each beat.
