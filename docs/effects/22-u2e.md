# Part 22 — U2E ("Vector Part II" — 3D City Flyover)

## Overview

The climactic 3D scene: a camera flies through a procedural city of
flat-shaded and Gouraud-shaded buildings, trees, tunnels, and a
spaceship, driven by a pre-baked animation sequence. Uses the full
Future Crew "U2" 3D engine.

**Duration:** ~510.4 s – 563.7 s  
**Frame rate:** 35 fps (70 Hz / 2)  
**Author:** PSI

## Architecture

### U2 3D Engine (`u2engine.js`)

Self-contained module that implements:

1. **Data loading** — Parses the FC binary object format (VERT, NORM,
   POLY, ORD0/ORDE chunks) into JS objects. Loads scene (00M),
   42 object files, and animation data (0AB).
2. **Animation decoder** — Bytecode interpreter that updates object
   visibility, translation, and rotation matrices each frame. Handles
   FOV changes and end-of-animation markers.
3. **3D pipeline** — Per-object: apply camera matrix, rotate normals,
   project vertices to 2D with clipping flags.
4. **Object sorting** — Painter's algorithm: bubble-sort objects by
   distance. Special-case floor objects (always drawn first) and
   the spaceship (forced last in a specific frame range).
5. **Polygon drawing order** — Each object stores multiple pre-computed
   polygon lists; the one whose orientation point is nearest camera is
   chosen.
6. **Backface culling** — Dot product of polygon normal and view vector.
7. **Polygon clipping** — Sutherland-Hodgman style against Z near-plane,
   top, bottom, left, right edges.
8. **Flat shading** — Directional light dot-product mapped to palette
   color offset.
9. **Gouraud shading** — Per-vertex light interpolated along edges and
   scanlines.
10. **Polygon filling** — Zone-based scanline fill for both flat and
    Gouraud modes.

### Data (`data.js`)

- **U2E_00M_base64** — Scene file: palette + object list.
- **U2E_0AB_base64** — Animation bytecode.
- **U2E_001..042** — 42 binary 3D object files (buildings, trees,
  tunnels, roads, spaceship, etc.).

### Effect (`effect.js`)

Handles four phases:

1. **Fade to white** (frames 0–32 @70Hz) — Transition from previous part.
2. **Transition setup** (frames 33–48) — White frame with border zones
   using palette entries 252/253/254.
3. **Reveal** (frames 49–80) — Cross-fade border colors to reveal the
   3D viewport area.
4. **3D animation** (frame 81+) — Steps the animation, clears viewport,
   renders the scene via the U2 engine, displays indexed framebuffer.

Final fade to white is handled when the animation ends.

## Scrubbing

The animation is stateful (cumulative matrix updates). For scrubbing:
- Engine state is cached every 100 animation frames.
- Replay starts from the nearest cached checkpoint.
- Cache is capped at 50 entries to bound memory usage.

## Remastered Ideas

- Full 4K with anti-aliased polygon edges.
- Real-time shadow mapping for buildings.
- Normal-mapped surfaces and specular highlights.
- Volumetric fog/haze for depth.
- Reflective windows on buildings.
- Post-processing bloom on bright palette entries.
- Smooth camera interpolation instead of stepped animation.
