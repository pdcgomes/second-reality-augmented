# Techno Circles Remastered — Overview

**Effect:** Part 8 of the Second Reality demo (1993, Future Crew)
**What you see:** Two orbiting circle patterns creating moiré interference, with per-scanline distortion
**Duration:** 17.7 seconds (168.1 s – 185.8 s in the full demo timeline)

---

## What This Guide Covers

This guide unpacks the remastered TECHNO_CIRCLES effect layer by layer. By the
end you will understand:

- How EGA bit-plane circle bitmaps encode concentric ring patterns in packed bits
- How quarter-circle mirroring reconstructs a full 640×400 circle from a 320×200 quadrant
- How bitwise OR of two circle patterns produces a 16-colour moiré interference
- How sinusoidal orbits move the circles on independent paths
- How per-scanline horizontal distortion creates wave-like warping that intensifies over time
- How the rendering pipeline converts palette indices to themed colours with bloom

Each layer builds on the last. You can stop at any point and still have learned
something valuable.

## File Map

The effect lives in three source files plus shared WebGL helpers:

```
src/effects/technoCircles/
  effect.js               — Classic variant (251 lines)
                            CPU software renderer, 320×200 VGA palette.
                            Faithful 1:1 recreation of the 1993 original.

  effect.remastered.js    — Remastered variant (652 lines)
                            GPU-accelerated circle interference using
                            the original EGA bitmaps as textures and
                            GLSL for palette lookup, distortion, and bloom.

  data.js                 — Circle bitmaps (3 lines)
                            Base64-encoded EGA bit-plane data for both
                            circle patterns. Generated by extraction tool.

src/core/
  webgl.js                — Shared WebGL2 helpers (90 lines)
                            createProgram(), createFullscreenQuad(), etc.
```

## Architecture

Both variants decode the same pair of EGA circle bitmaps at init time. The
classic renders on the CPU into a 320×200 framebuffer using palette lookups.
The remastered uploads the decoded bitmaps as GPU textures and computes
interference, palette mapping, distortion, and bloom entirely in GLSL.

```
              data.js
       (circle1 + circle2 bitmaps,
        base64-encoded EGA planes)
               /           \
              v             v
       effect.js      effect.remastered.js
    (CPU rasterise)   (GPU texture sample + GLSL)
     320×200 VGA       native resolution
     palette lookup    procedural palette in shader
     no post-fx        bloom + beat reactivity
```

The key architectural insight: all animation state — orbital positions,
distortion intensity, palette cycling — is derived purely from the frame
number. There is no persistent state. Jump to any point in time and the
effect computes the correct frame from scratch. This makes scrubbing in the
editor trivial.

## The Effect Interface

Every effect in this project exports three methods:

```javascript
export default {
  init(gl) { },             // called once — set up shaders, buffers, textures
  render(gl, t, beat, params) { },  // called every frame
  destroy(gl) { },          // called when the effect is unloaded
}
```

Where:
- `gl` — a WebGL2 rendering context
- `t` — seconds elapsed since this clip started (not global time)
- `beat` — 0.0 to 1.0 position within the current musical bar
- `params` — key-value object of tunable parameters from the editor

The remastered variant also exports a `params` array describing 10 tunable
knobs (palette theme, colour smoothing, bloom strength, etc.) that the editor
auto-generates UI controls for.

## Two Phases

The effect runs in two distinct visual phases:

| Phase | Frames | Duration | Visual |
|-------|--------|----------|--------|
| Phase 1 | 0–255 | ~3.7 s | Single circle, rotating palette sweep fading in |
| Phase 2 | 256+ | ~14.0 s | Two-circle interference with growing distortion |

Phase 1 introduces the circle pattern gently — a single centred circle with
a colour palette that fades in and rotates. Phase 2 is the main show: two
circles on independent sinusoidal orbits overlap to create moiré interference,
with a per-scanline sinusoidal distortion that starts at zero and gradually
intensifies, warping the patterns into organic wave shapes.
