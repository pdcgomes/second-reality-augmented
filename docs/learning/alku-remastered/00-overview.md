# Alku Remastered — Overview

**Effect:** Part 1 of the Second Reality demo (1993, Future Crew)
**What you see:** Opening credits fading in/out over black, then over a scrolling landscape
**Duration:** ~62 seconds (0 s – 62 s in the full demo timeline)

---

## What This Guide Covers

This guide unpacks the remastered ALKU (opening credits) effect layer by layer.
By the end you will understand:

- How a pre-rendered 640×200 landscape image scrolls smoothly via GPU texture sampling
- How a variable-width **bitmap font** with 4 anti-aliasing levels is parsed and rendered
- How **palette-based fading** from the original VGA era maps to shader uniforms
- How the GPU composites landscape, text, horizon glow, and bloom in multiple passes
- How to extend the effect with your own credits, fade curves, and parallax layers

Each layer builds on the last. You can stop at any point and still have learned
something valuable.

## File Map

The effect lives in three source files plus shared WebGL helpers:

```
src/effects/alku/
  data.js                 — Base64-encoded assets (~228 KB)
                            Landscape (640×200), font (1500×30), VGA palette.
                            Generated by tools/extract-alku-data.mjs.

  effect.js               — Classic variant (470 lines)
                            Single-pass compositing shader. Faithful 1:1
                            recreation of the 1993 original.

  effect.remastered.js    — Remastered variant (693 lines)
                            Multi-pass GPU pipeline with horizon glow,
                            dual-tier bloom, beat-reactive brightness.

src/core/
  webgl.js                — Shared WebGL2 helpers
                            createProgram(), createFullscreenQuad(), etc.
```

---

## Architecture

Both variants decode the same base64 assets from `data.js` at init time. The
classic renders everything in a single fullscreen shader pass. The remastered
adds a scene FBO, bloom pipeline, and atmospheric effects while keeping the
original pixel-art character through NEAREST-neighbor filtering.

```
                 data.js
        (landscape, font, palette)
               /           \
              v             v
       effect.js      effect.remastered.js
    (single pass)     (multi-pass pipeline)
     320×256 fixed     native resolution
     no post-fx        bloom + horizon glow
     no params         7 editor parameters
```

The key architectural insight: both variants run the same `computeState(t)`
sequence logic, so credits appear at identical times. You can toggle between
classic and remastered and the text timing matches perfectly.

## The Opening Credits Sequence

The effect is a **timed sequence of 9 phases**, each triggered at a pre-authored
timestamp synchronised to the soundtrack:

```
 0.0s ─── Black wait ──────────────────────────────────────── 1.5s
 1.5s ─── "A / Future Crew / Production" (over black) ────── 8.5s
 8.5s ─── "First Presented / at Assembly 93" (over black) ── 15.5s
15.5s ─── "in / [Dolby Logo]" (over black) ───────────────── 22.0s
22.0s ─── Landscape fade-in (no text) ────────────────────── 30.0s
30.0s ─── "Graphics / Marvel / Pixel" (over landscape) ───── 38.0s
38.0s ─── "Music / Purple Motion / Skaven" (over landscape)─ 46.0s
46.0s ─── "Code / Psi / Trug / Wildfire" (over landscape) ── 54.0s
54.0s ─── "Additional Design / Abyss / Gore" (over scroll) ─ 62.0s
```

Each text screen **fades in** over 64 frames (~0.9 s), **holds**, then
**fades out** over 64 frames. Text over black holds for 300+64 frames; text
over the scrolling landscape holds for 200+64 frames.

## The Effect Interface

Every effect in this project exports three methods:

```javascript
export default {
  init(gl) { },             // called once — decode assets, compile shaders
  render(gl, t, beat, params) { },  // called every frame
  destroy(gl) { },          // called when the effect is unloaded
}
```

Where:
- `gl` — a WebGL2 rendering context
- `t` — seconds elapsed since this clip started (not global time)
- `beat` — 0.0 to 1.0 position within the current musical bar
- `params` — key-value object of tunable parameters from the editor

The remastered variant also exports a `params` array describing 7 tunable
knobs (horizon glow, bloom strength, beat reactivity, etc.) that the editor
auto-generates UI controls for.
